01 — Overview

What is Amazon S3?

Amazon Simple Storage Service (S3) is an object storage service that offers industry-leading scalability, data availability, security, and performance. It stores data as objects inside buckets.

Unlike traditional file systems, S3 uses a flat namespace — every object has a unique key within a bucket. There are no actual folders; the slash "/" in key names is just a naming convention.

S3 can store any type of data — images, videos, backups, logs, static websites, ML datasets — with no upper limit on total storage.

🪣 my-bucket
├── 📁 images/
├── logo.png key: images/logo.png
└── banner.jpg key: images/banner.jpg
├── 📄 index.html key: index.html
└── 📦 backup.zip key: backup.zip
Every object = Key + Data + Metadata
02 — Core Concepts

Building Blocks of S3

🪣

Buckets

Containers for objects. Bucket names must be globally unique across all AWS accounts. Each bucket lives in a specific AWS Region.

s3://your-bucket-name/
📦

Objects

The fundamental entity in S3. An object = Key (name) + Value (data) + Metadata + optional Version ID. Max size: 5 TB.

Max single PUT: 5 GB
🔑

Keys

The unique identifier for an object within a bucket. Think of it as the "path". Prefix + delimiter simulate folder structures.

logs/2025/april/app.log
📋

Metadata

Name-value pairs attached to objects. System metadata (Content-Type, ETag) and user-defined metadata (x-amz-meta-*) up to 2 KB.

Content-Type: image/png
🕰

Versioning

Keep multiple versions of an object. Protects against accidental overwrites and deletes. Once enabled, can only be suspended — not disabled.

VersionId: abc123xyz
⚡

Multipart Upload

Upload large objects in parts (minimum 5 MB each). Recommended for files over 100 MB. Parts upload in parallel — faster, resumable.

aws s3 cp --part-size 100MB
03 — Storage Classes

Choose the Right Storage Class

S3 offers a range of storage classes designed for different access patterns and cost requirements.

Class Use Case Availability Min Duration Retrieval
S3 Standard Frequently accessed data 99.99% None Immediate
Intelligent-Tiering Unknown access patterns 99.9% None Immediate
Standard-IA Infrequent, rapid access 99.9% 30 days Immediate
One Zone-IA Non-critical infrequent data 99.5% 30 days Immediate
Glacier Instant Archives, quarterly access 99.9% 90 days Milliseconds
Glacier Flexible Long-term backups 99.99% 90 days Minutes–hours
Deep Archive 7–10 year retention 99.99% 180 days Up to 12 hrs
💡 Tip: Use S3 Lifecycle Rules to automatically transition objects between storage classes based on age — e.g., move to Standard-IA after 30 days, Glacier after 90 days.
04 — Security

Securing Your S3 Data

🔐

Bucket Policies

JSON-based resource policies attached to a bucket. Control who can do what — across accounts or for public access. More powerful than ACLs.

👤

IAM Policies

Attach to users, roles, or groups. Use identity-based policies for fine-grained per-user access. Always follow least-privilege principle.

🚫

Block Public Access

Account-level or bucket-level setting. Prevents any public ACL or bucket policy from making data public. Enabled by default for new buckets.

🔒

Encryption

SSE-S3 (AWS managed keys), SSE-KMS (your KMS key), SSE-C (customer-provided), or client-side encryption. All new objects encrypted by default since 2023.

🔗

Presigned URLs

Grant time-limited access to private objects without changing bucket permissions. Useful for temporary file sharing or upload links.

📡

VPC Endpoints

Access S3 privately from your VPC without traversing the public internet. Use Gateway Endpoints (free) or Interface Endpoints (PrivateLink).

Frontend
02

Data Lake

Store structured/unstructured data at scale. Query directly with Athena (SQL) or process with EMR / Glue.

Analytics
03

Backup & Disaster Recovery

Cross-region replication, versioning, and Object Lock (WORM) for compliance. 99.999999999% durability means your data is safe.

DevOps
04

Application Storage

Store user uploads, media files, and documents. Trigger Lambda on S3 events for image processing, transcoding, or indexing.

Backend
05

CI/CD Artifact Storage

Store build artifacts, deployment packages, and IaC state files (Terraform remote backend uses S3 + DynamoDB for state locking).

CI/CD
06

Centralize ALB, CloudFront, CloudTrail, and VPC Flow Logs in S3. Use lifecycle rules to archive to Glacier and cut storage costs.
